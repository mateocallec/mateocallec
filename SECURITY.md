## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

If you discover a vulnerability, I ask that you follow responsible disclosure practices as outlined in my **[Vulnerability Disclosure Policy](https://mateocallec.com/vulnerability-disclosure)**.

### How to report

You can report a vulnerability through one of the following channels:

- **GitHub Private Security Advisory** *(preferred)*: Use the [Report a vulnerability](../../security/advisories/new) button in the Security tab of this repository.
- **Email**: [mateo@callec.net](mailto:mateo@callec.net?subject=%5BSECURITY%5D%20%5BPROJECT%20NAME%5D%20-%20Brief%20description) — Please use the subject line `[SECURITY] [PROJECT NAME] - Brief description`.

### What to include

To help me triage and reproduce the issue efficiently, please provide:

- A clear description of the vulnerability and its potential impact
- Affected version(s) or commit(s)
- Step-by-step reproduction instructions
- Proof-of-concept code or screenshots (if applicable)
- Any suggested remediation (optional but appreciated)

---

## Response Timeline

I will make every reasonable effort to:

| Step | Target timeframe |
|------|-----------------|
| Acknowledge receipt of your report | Within **72 hours** |
| Confirm and assess the vulnerability | Within **7 days** |
| Release a patch or mitigation | Within **90 days** (depending on severity and complexity) |
| Publicly disclose the issue | After a fix is available and deployed |

If you do not receive a response within 72 hours, feel free to follow up.

---

## Scope

This policy applies to security vulnerabilities found in the source code of this repository. The following are **out of scope**:

- Vulnerabilities in third-party dependencies (please report those upstream)
- Issues already known or previously reported
- Attacks requiring physical access to a device
- Social engineering or phishing attempts

---

## Legal

I will not pursue legal action against researchers who:

- Act in good faith and follow this policy
- Do not access, modify, or delete data beyond what is necessary to demonstrate the vulnerability
- Do not disclose the vulnerability publicly before a fix is available
